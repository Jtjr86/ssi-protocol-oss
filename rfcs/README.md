## RFC Categories (Informational Only)

### **Informational RFC**
Documents describing SSI-related patterns, use cases, or ecosystem observations. No protocol changes proposed.

### **Experimental RFC**
Proposes experimental extensions or alternative approaches for community evaluation. Not binding on protocol evolution.

### **Standards Track** (FUTURE)
Reserved for when formal governance exists. Not accepted currently.

## Submission Process

### 1. Fork Repository
Fork the SSI Protocol repository on GitHub.

### 2. Create RFC File
Copy `rfcs/0000-template.md` to `rfcs/0000-your-title.md`

### 3. Fill Template
Complete all required sections with technical detail.

### 4. Submit Pull Request
Open PR with title: `RFC: [Your Title]`

Label: `rfc-informational` or `rfc-experimental`

### 5. Community Discussion
Discussion happens in PR comments. No formal review process exists.

### 6. Archival
Accepted PRs are merged into `rfcs/archive/` directory for historical record. This is **not approval** ‚Äî only acknowledgment of receipt.

This RFC intake system serves three purposes:

3. **Preparation:** Create foundation for future formal governance

- Create governance authority
